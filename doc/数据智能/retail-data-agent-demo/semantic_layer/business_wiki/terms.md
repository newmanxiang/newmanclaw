# Terms

## 销售额

- 默认等同于 GMV
- 必须扣减退款：`pay_amount - refund_amount`
- 不使用商品标价 `list_price` 直接求和

## 销售额下降

- 判定方式：当前周期 GMV 低于对比周期 GMV
- 默认对比周期：上一等长周期
- 最小输出：当前周期 GMV、对比周期 GMV、差额、降幅
- 降幅公式：`(current_gmv - previous_gmv) / previous_gmv`

## 最近7天

- 判定方式：以数据中最大 `orders.order_date` 为结束日期，向前取7个自然日
- 示例：最大日期为2026-05-10时，最近7天为2026-05-04至2026-05-10
- 原因：演示数据不是实时数据，不能使用系统当天日期

## 上一等长周期

- 判定方式：与当前周期长度相同，且紧邻当前周期之前
- 示例：当前周期为2026-05-04至2026-05-10时，对比周期为2026-04-27至2026-05-03

## 下降原因拆解

- 默认顺序：一级品类 -> 门店 -> 渠道
- 贡献判断：优先找 `delta_gmv` 最负的维度成员
- 输出时区分"主要贡献因素"和"伴随现象"
